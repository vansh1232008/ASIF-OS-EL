What is CPU Scheduling?

Definition

CPU Scheduling is a process of determining which process will own the CPU for execution while another process is on hold. The main task of CPU scheduling is to ensure that whenever the CPU remains idle, the OS selects one of the processes available in the ready queue for execution. The selection process is carried out by the CPU scheduler, which selects one of the processes in memory that are ready for execution.

Why CPU Scheduling?

A typical process involves both I/O time and CPU time. In a uniprogramming system like MS-DOS, time spent waiting for I/O is wasted, and the CPU remains idle during this time. In multiprogramming systems, one process can use the CPU while another is waiting for I/O, which is possible only with effective process scheduling.

    Objectives of Process Scheduling Algorithm:
  • Maximize CPU utilization [Keep CPU as busy as possible]
  • Fair allocation of CPU
  • Maximize throughput [Number of processes that complete their execution per time unit]
  • Minimize turnaround time [Time taken by a process to finish execution]
  • Minimize waiting time [Time a process waits in the ready queue]
  • Minimize response time [Time when a process produces its first response]

Important Terminologies

  • CPU Burst Time: The time required by a process for its execution on the CPU.
  • Waiting Time: The total time a process has been in the ready queue waiting for CPU allocation.
  • Turnaround Time: The total time taken from the submission of a process to its completion.
  • Response Time: The time from the submission of a request until the first response is produced.

Types of CPU Scheduling Algorithms

  • First-Come, First-Served (FCFS): The simplest scheduling algorithm where the process that arrives first gets executed first.
  • Shortest Job Next (SJN): Selects the process with the smallest execution time.
  • Priority Scheduling: Each process is assigned a priority, and the process with the highest priority is executed first.
  • Round Robin (RR): Each process is assigned a fixed time slice in a cyclic order.
  • Multilevel Queue Scheduling: Processes are divided into different queues based on priority or type, and each queue has its own scheduling algorithm.

Scheduling Criteria

Different scheduling algorithms aim to optimize various criteria that define the "goodness" of the scheduling technique. Some key criteria include:

  • CPU Utilization: Ensuring the CPU is kept as busy as possible by minimizing idle time.
  • Throughput: Maximizing the number of processes that complete their execution in a given time frame.
  • Turnaround Time: Minimizing the total time taken for a process from submission to completion.
  • Waiting Time: Reducing the time a process spends in the ready queue before execution starts.
  • Response Time: Shortening the time from request submission until the first output is produced, crucial for interactive systems.
  • Fairness: Ensuring no single process faces indefinite postponement and that all processes get a fair share of CPU time.

Real-Time Scheduling

Real-time systems require that tasks be completed within strict timing constraints, often called deadlines. Unlike general-purpose systems, where average performance metrics like throughput and waiting time are important, real-time scheduling focuses on predictability and meeting these critical deadlines:

  • Hard Real-Time Systems: Missing a deadline could lead to catastrophic consequences, so tasks must be guaranteed to complete by their deadlines.
  • Soft Real-Time Systems: Deadlines are important, but occasional misses may be tolerable, though they degrade system performance.
  • Common Real-Time Scheduling Algorithms:
    • Rate Monotonic (RM): A static priority algorithm where tasks with shorter periods get higher priority.
    • Earliest Deadline First (EDF): A dynamic priority algorithm where tasks closest to their deadlines have the highest priority.
    • Least Laxity First (LLF): Prioritizes tasks with the least "slack" time, focusing on tasks running out of time faster.

Real-time scheduling ensures that time-critical processes receive the necessary processor time to maintain system correctness and reliability.
